Search

Search problems are dynamic problems with an infinite horizon where the agent has to decide to:

  • "Should I stay or should I go?"—The Clash (1981).
  • Should I accept the current job offer or keep searching for better jobs?
  • Should I settle down or keep searching for a life partner?
  • Should I buy plane tickets now or keep searching for better deals?
  • Should I enter this market now or delay entry?
  • Should I exit this market now or delay exit?
  • Should I stop tuning and deploy this model or keep searching for better parameters?
  • Should I attack now or defend and wait for an opportunity of defeating the enemy?—Sun Tzu, The Art of War

A Model

  • At each period $t$, the agent chooses $a_t =$ stop $s$ or continue $c$.
  • There is a set of states $S$ and the probability of the state in period $t$ being equal to $x_t$ given past state realizations satisfies: $$\Pr(s_t = x_t \mid s_{t-1} = x_{t-1}, s_{t-2} = x_{t-2}, \ldots) = \Pr(s_t = x_t \mid s_{t-1} = x_{t-1}) = q(x_t \mid x_{t-1})$$ Note: "states" here are not the same as "states" in the previous sections.
  • In each period $t$, provided the agent did not stop yet, the agent gets utility $u(s_t, a_t)$—it depends only on the state and the action, not on calendar time. If the agent chooses to stop, $a_t = s$, the agent receives $u(s_t, s)$ at period $t$ and zero in future periods.
  • The agent maximizes discounted expected utility: $$\mathbb{E}\left[\sum_{t=0}^{\infty} \delta^t \, u(s_t, a_t)\right]$$

The Value Function

Notice that the value function will depend only on the current state but NOT on the calendar time. The value function must satisfy the recursive equation:

$$V(s_t) = \max\left\{ u(s_t, s), \; u(s_t, c) + \delta \sum_{s \in S} V(s) \cdot q(s|s_t) \right\} \quad\textcolor{#c17900}{(\text{R})}$$

A Tree Cutting Problem

Fudenberg & Tirole, Game Theory (1991)

A landowner has a tree whose timber value $v(t)$ grows with age $t$, but at a diminishing rate ($v(0) = 0$, $v' \gt 0$, $v'' \lt 0$). At each period $t$, the owner decides to cut the tree ($s$) or to wait ($w$):

  • If the owner cuts: $u(t, s) = v(t)$
  • If the owner waits: $u(t, w) = 0$

The transition is deterministic: state $t$ transitions to $t + 1$ with certainty.

The value function satisfies:

$$V(t) = \max\left\{ v(t), \; \delta \cdot V(t+1) \right\}$$

A Tree Cutting Problem

Continuation

If $t^\ast$ is the optimal stopping time:

Cutting earlier is not profitable: at $t^\ast - 1$, waiting yields more than cutting:

$$v(t^\ast - 1) \leq \delta \cdot v(t^\ast)$$

Cutting later is not profitable: cutting at $t^\ast$ yields more than waiting:

$$v(t^\ast) \geq \delta \cdot v(t^\ast + 1)$$

Online Shopping

An Example

A consumer is searching to buy one unit of a good online. Let $p_t$ denote the price at time $t$.

  • If the consumer buys (stops): $u(p_t, s) = v - p_t$
  • If the consumer keeps searching (continues): $u(p_t, c) = 0$

The price increases by a factor of $\kappa \gt 1$ or decreases by a factor of $1/\kappa$, each with equal probability:

$$q(\kappa p \mid p) = q(p/\kappa \mid p) = \tfrac{1}{2}$$

Notice that any price is of the form $p_t = \kappa^z \cdot p_0$ where $z$ is an integer. Thus, for simplicity, we use the set of integers $\mathbb{Z}$ as the set of states.

Online Shopping

An Example (continuation)

It can be shown that any optimal strategy will be a threshold strategy:

$$a^z(s_t) = \begin{cases} s & \text{if } s_t \leq z \\ c & \text{otherwise} \end{cases}$$

Our goal is then to find the value of $z$ such that the recursive equation $\textcolor{#c17900}{(\text{R})}$ is satisfied.

Let $V^z(s_t)$ be the value of state $s_t$ if the agent follows the strategy $a^z$. Notice that $V^z$ might not be the value function. The function $V^z$ satisfies some recursive equation but it might not satisfy the recursive equation $\textcolor{#c17900}{(\text{R})}$.

$$V^z(s) = \begin{cases} v - p_0 \kappa^s & \text{if } s \leq z \\ \frac{\delta}{2} \cdot \left( v - p_0 \cdot \kappa^z + V^z(s+1) \right) & \text{if } s = z+1 \\ \frac{\delta}{2} \cdot \left( V^z(s-1) + V^z(s+1) \right) & \text{if } s \gt z+1 \end{cases} \quad (\text{Z})$$

An Example (solution)

$$V^z(s) = \begin{cases} v - p_0 \kappa^s & \text{if } s \leq z \\ (v - p_0 \kappa^z) \cdot r^{s-z} & \text{if } s \gt z \end{cases}$$

where $0 \lt r = \dfrac{1 - \sqrt{1 - \delta^2}}{\delta} \leq 1$.

Let $z^\ast$ be the optimal $z$, then it must satisfy the One-shot Deviation Principle we shall see next.

One-Shot Deviation Principle

Definition: One-Shot Deviation
Let $a = (a_t)$ be some search policy. The alternative policy $\tilde{a} = (\tilde{a}_t)$ is a one-shot deviation at period $t$ and state $s_t$ if $\tilde{a}_t(s_t) \neq a_t(s_t)$ but $\tilde{a}_k(s) = a_k(s)$ for all future periods $k \gt t$ and all states $s$.
Definition: Profitable One-Shot Deviation
A one-shot deviation $\tilde{a}$ from policy $a$ at period $t$ and state $s_t$ is profitable if it yields a strictly higher expected discounted utility than $a$ at $s_t$ where $\tilde{a}_t(s_t) \neq a_t(s_t)$.
The One-Shot Deviation Principle
A policy $a^\ast = (a_t^\ast)$ is optimal if and only if there is no alternative policy $\tilde{a}$ that is a profitable one-shot deviation.

The One-Shot Deviation Principle in Action

If the value of $z$ is optimal, the One-Shot Deviation Principle requires two conditions at the boundary:

At $s = z$ (agent stops): deviating to continue should not be profitable:

$$v - p_0\kappa^{z} \geq \frac{\delta}{2}\left(V^{z}(z - 1) + V^{z}(z + 1)\right)$$

At $s = z + 1$ (agent continues): deviating to stop should not be profitable:

$$\frac{\delta}{2}\left(V^{z}(z) + V^{z}(z + 2)\right) \geq v - p_0\kappa^{z+1}$$

Define $f(z) = v - p_0 \kappa^z$ and $g(z) = \frac{\delta}{2}\left(V^z(z) + V^z(z+2)\right)$, for the optimal $z$:

$$f(z) \geq g(z-1) \geq g(z) \geq f(z+1)$$
